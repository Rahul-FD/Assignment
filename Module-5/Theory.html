1.What are the new tags added in HTML5?

Ans.

HTML5 introduced several new tags compared to previous versions of HTML. Some of the notable new tags added in HTML5 include:
1.<header>: Defines a header for a document or section.
2.<nav>: Defines a container for navigation links.
3.<article>: Represents an independent piece of content within a document, such as a blog post or news article.
4.<section>: Defines a section of a document, often used in conjunction with the <article> tag.
5.<aside>: Represents content that is tangentially related to the content around it, such as sidebars or pull quotes.
6.<footer>: Defines a footer for a document or section.
7.<main>: Specifies the main content of a document, excluding headers, footers, and navigation.
8.<figure>: Represents self-contained content, such as images, diagrams, or code snippets, with an optional caption provided by the <figcaption> tag.
9.<figcaption>: Provides a caption or description for content within a <figure> element.

These are just some of the key additions introduced in HTML5. There are other new elements and attributes as well, which collectively aim to enhance the structure, semantics, and functionality of web documents.

2.How to embed audio and video in a webpage?

Ans.

To embed audio and video in a webpage using HTML5, you can use the <audio> and <video> elements, respectively. Here's how you can do it:

  • Audio


    • 1.<audio>: This is the container element for embedding audio content.
    2.controls: This attribute adds playback controls (play, pause, volume, etc.) to the audio player.
    3.<source>: This element specifies the audio file to be played and its type.
    4.src: Specifies the URL of the audio file.
    5.type: Specifies the MIME type of the audio file.


  • Vedio


    • 1.<video>: This is the container element for embedding video content.
    2.controls: This attribute adds playback controls (play, pause, volume, etc.) to the video player.
    3.width and height: These attributes specify the dimensions of the video player.
    4.<source>: This element specifies the video file to be played and its type.
    5.src: Specifies the URL of the video file.
    6.type: Specifies the MIME type of the video file.

    3.Semantic element in HTML5?

    Ans.

    Semantic elements in HTML5 are elements that carry meaning and convey the structure of the content, rather than just being presentational. They provide a clearer indication of the purpose or function of the content they contain. Using semantic elements properly can improve accessibility, search engine optimization (SEO), and the overall organization of a webpage. Some of the semantic elements introduced in HTML5 include:
    1.<header>: Represents introductory content typically at the top of a page or section, often containing headings, logos, navigation, etc.
    2.<nav>: Defines a container for navigation links within a webpage.
    3.<section>: Represents a thematic grouping of content within a document, such as chapters, headings, or groups of related content.
    4.<article>: Represents an independent, self-contained piece of content that can be distributed and reused independently from the rest of the document, such as a blog post or news article.
    5.<aside>: Represents content that is tangentially related to the content around it, often used for sidebars or additional information.
    6.<footer>: Represents a footer for a section or the whole document, typically containing copyright information, contact details, or links to related resources.
    7.<main>: Defines the main content of a document, excluding headers, footers, and navigation.
    Using these semantic elements appropriately helps both developers and browsers understand the structure and meaning of a webpage's content, leading to better accessibility, maintainability, and search engine ranking.

    4.Canvas and SVG tags.

    Ans.

    <canvas> Element:
    The <canvas> element is a resolution-dependent bitmap canvas, which means it is essentially a rectangular area where you can draw graphics using JavaScript. It's suitable for rendering dynamic and interactive graphics, such as games, data visualization, or complex animations.

    <svg> Element:
    SVG (Scalable Vector Graphics) is an XML-based vector image format that allows you to define graphics using XML syntax. It's resolution-independent and ideal for creating static or dynamic graphics that can be scaled without losing quality. SVG is well-suited for diagrams, logos, icons, and other scalable graphics.
